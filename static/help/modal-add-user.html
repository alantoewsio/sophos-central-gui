Add user

Administrator workflow launched from Settings → Users.

Introduction

This nested dialog creates another local portal account. It layers on top of the Settings modal until you submit or close it.

Form fields

  • Username — unique identifier for sign-in.
  • Full name, Email, Mobile — optional contact metadata.
  • Role — User or Admin, controlling access to credentials and destructive tools.
  • Password — initial secret meeting minimum length rules.

After creation, the new row appears in the users table and can sign in immediately.
