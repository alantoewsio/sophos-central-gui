Settings — Update

Optional git-based automatic fast-forward updates for development checkouts.

Introduction

Administrators can set how often the server checks the default remote for your current branch. This panel only appears when git is installed and the app is running from a git working tree (typically a source checkout).

What it does

On each scheduled run the service runs git fetch, compares your HEAD with the configured upstream, and runs git pull --ff-only when your branch is behind. Diverged branches, merge conflicts, or unpushed work are left untouched; you will see a short status message in Settings after a check. After code or static files change, restart the app so the new files are loaded.

Availability

PyInstaller bundles and installs without a .git directory do not show this section. The maximum scheduled interval is 30 days; choosing Never disables background checks.
