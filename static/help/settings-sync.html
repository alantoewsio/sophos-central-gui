Settings — Sync

Scheduler configuration per Central credential.

Introduction

The Sync table lists every stored credential with separate schedules for full and incremental sync. Full sync pulls the complete Central snapshot into the local SQLite database; incremental sync refreshes firewalls and alerts more frequently. Background jobs authenticate and update the data the UI reads.

Intervals

Edit full and incremental intervals to balance freshness against API rate limits and server load. Shorter incremental intervals keep alerts and firewall list changes tighter without repeating the full catalog import on every poll. Set incremental to None to disable that schedule while keeping full sync.

Manual sync

The circular sync icon on a row runs a full sync only, then updates whoami-style metadata after a successful token exchange. Failures surface in status text and server logs for troubleshooting.
