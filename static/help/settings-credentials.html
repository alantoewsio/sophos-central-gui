Settings — Central credentials

OAuth-style API clients used to call Sophos Central.

Introduction

Each credential row pairs a display name with the client ID and secret issued in Sophos Central. The scheduler uses these entries to keep tenants, firewalls, alerts, and licenses current.

Secret storage

Client secrets are encrypted at rest on the application host. Back up the secrets database according to your security policy; anyone with the files and keys could impersonate your integration.

Verification

Before saving a new credential, use Test connection to confirm Central accepts the client. The UI stores metadata such as the resolved Central account identifier when validation succeeds.

Maintenance actions

Icons on each row may let you edit the friendly name, rotate verification, remove the credential, or jump to sync options. Removing a credential stops future syncs for that client until you add a replacement.
